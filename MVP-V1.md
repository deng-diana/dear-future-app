# Reunite — MVP V1 范围(已钉死 2026-06-09)

> 灵魂见 [MANIFESTO.md](MANIFESTO.md)。这份只说 V1 到底做什么。
>
> 一句话:**写一封纯文字的信,选一个至少 15 天后的日子,封存;封存即从 app 消失;到那天,信放生到你的邮箱。**

## 用户旅程(整个 app 几乎只有一屏:写信)

1. 打开 → 一块空白的写信屏(第一次进,有一行话说明这是什么)。
2. 写信(纯文字,写的时候随便改)。
3. 选一个未来的日期(**最短 15 天后**,不允许更近 —— 没有时间,就没有遗忘)。
4. 封存。第一次封存时问:"这封信将来寄到哪个邮箱?" → 输入 → 验证(OTP / 魔法链接)。
   这个邮箱 = 你的账号 + 将来放生的地址。
5. 封存完成 → **信从 app 彻底消失**:没有列表、没有计数、没有红点、没有"在路上"。
   屏幕回到空白,你唯一能做的是"再写一封"。
6. 到了那天 → 一封邮件到你的信箱:它把你领到一个**精心设计的「看信页」(网页)** 去拆封、读信(字体、配乐、将来的音视频都在那儿);邮件正文底部**还夹着信的纯文字**,万一链接将来打不开,字也丢不了。你不必打开 app。

## 做(In)

- 写信:纯文字
- 选送达日期:最短 +15 天
- 封存:封存即锁死(不可改、不可看)且从 app 消失
- 送达:到期当天发邮件到本人邮箱(= 放生)
- 账号:邮箱 + OTP / 魔法链接,无密码;设备保持登录,之后写信无需重复验证

## 不做(Out — 先忍住)

- 多媒体(视频 / 语音 / 照片)→ V2
- 寄给别人 → 以后再说
- 任何"我的信件"列表 / 收件箱 / 红点 / 统计 → **永不做(违背第一戒)**
- 重录 / 到期前反悔撤回 → 先不碰

## 几条铁规则

- **最短封存 15 天**:`deliver_on >= today + 15d`,前端拦、后端也拦。(下限定在 15 天:仍远大于"提醒类"备忘录,又给特殊情况留空间。代码里只在一处定义这个数字,改起来一秒钟。)
- **封存即锁死**:封存后内容不可编辑、不可查看。
- **封存即消失**:app 内不保留任何可见痕迹。
- **登录极简**:邮箱即账号;唯一目的 = 证明邮箱是你的 + 记住你让写信顺手。
  不存在"登录后的主页",因为没有列表可看。

## 数据模型(最小)

```
letters
  id            主键
  owner_email   身份 + 放生地址
  body          信的内容(静态加密存储)
  sealed_at     封存时间
  deliver_on    用户选的那天 (date)
  delivered_at  送达后回填(防重发);null = 未送达
```

(外加邮箱验证所需的最小 auth:OTP / 魔法链接 + 设备 session。)

## 送达机制(关键)

- **不要为每封信定一个未来闹钟。**
- 改为:**每天跑一次定时任务**,查 `deliver_on <= today AND delivered_at IS NULL` → 发邮件 → 回填 `delivered_at`。
- 幂等、抗重启、十年不变。

## 封存那一刻:仪式(宝盒是动词,不是名词)

- 火漆封印 → 放进时光宝盒 → **宝盒连同信一起消失**。
- 情绪是"**放手、交给时间**",不是"存进我的收藏夹"。
- 宝盒绝不能变成一个以后能打开、能看到"我攒了几封"的地方 —— 那是「列表」换皮,破第一戒。

## 重逢那一刻:「看信页」+ 信使邮件(灵魂的另一半,别敷衍)

**看信页(网页)= 看信的主舞台,当作 craftsmanship 好好雕。**
- 拆封 / 火漆裂开 / 信纸展开的动效在这里。
- 自定义字体、排版、配乐;将来的视频 / 语音 / 照片也都在这儿(邮件做不到这些)。
- 一个唯一、不可猜的链接直接打开,无需登录。

**信使邮件 = 把人领上舞台 + 一份保命副本。**
- 顶部:温柔的邀请 +「点此拆封 →」链接(主路径)。
- 底部:信的**纯文字全文**,作为"万一链接打不开"的保命副本。
- 发件人显示名:**「20XX 年的你」**;地址用我们认证过的发信域名(保证送达);Reply-To = 用户自己。
- 取舍:正文放全文会轻微"剧透"动效 → 把全文压到底部、弱化,主路径仍是点链接进看信页。**字到得了 > 仪式做得完美。**
- 长跨度提醒:封存那一刻就告诉用户"将来会收到一封署名'你自己'的信,别当垃圾邮件"。

## 看信页 · 体验设计原则(待实现,之后专门设计)

> 别处都在做减法;只有这一页做加法,是 craftsmanship 尽情燃烧的地方。

- **先定情绪,再定形式。** 用户点开链接的 0.5 秒内,最该"心头一颤"的是什么?
  候选:火漆裂开的那一声 / "20XX 年的你"几个字 / 信纸一寸寸展开……(待定其一为主角)
- **节奏:慢。** 拆封是"慢慢地、有呼吸地展开",不是"啪一下全开"。
  这一页的灵魂在"慢" —— 它要和外面那个快节奏的世界,正好相反。
- 待设计:主角情绪 / 拆封节奏曲线 / 字体 / 配乐 / 留白与呼吸。

## 第三戒的底线:我们绝不做"唯一的那根线"

- 不承诺"信永生",只承诺:**信的留存,绝不只依赖我们。**
- 文字(V1):纯文字永远躺在用户自己邮箱的正文里。
- 将来有音视频:用**附件**随邮件进用户邮箱,或推到**用户自己的云**。
- 看信页(网页)是最美的那层,但它依赖我们的服务器,所以**允许它将来消失** —— 它绝不能是唯一的副本。

## 隐私(V1 诚实声明)

- 信在数据库里 **静态加密**。
- 但因到期要靠我们替你发邮件,**做不到"连我们都解不开"(非 E2E)**。V1 如实告知,不假装。
- (V2 可探索客户端加密,但会与邮件送达冲突,先放着。)
