# Reunite — PRD · 写给未来的自己(2 天黑客松版)

> 灵魂见 [MANIFESTO.md](MANIFESTO.md)。范围见 [MVP-V1.md](MVP-V1.md)。
> 这份 PRD 只回答一件事:**两天里,做出能完整演示的那个闭环。**
>
> PRD = Product Requirements Document = 产品需求文档(说清楚"要做成什么样")。
> 文档用中文写给你看;**app 里给用户看的字一律用英文**(产品面向全球)。

---

## 0. 一句话

写一封信 → 选一个未来的日子 → 封存(信从 app 消失)→ 到那天,它作为一封邮件回到你自己的邮箱。

## 1. 北极星:这一刻该有什么感觉

不是"我存了个备忘录",是 **"我把此刻的自己,亲手交给了时间"**。
每一处文案、每一个动效,都要服务这一种感觉:**郑重、安静、放手**。
违背这感觉的(列表、红点、计数、催促),再方便也不做。

---

## 2. 黑客松范围:做 / 不做 / 假装

| 做(In) | 不做(Out) | 演示时"假装"(Faked) |
|---|---|---|
| 写信(纯文字) | 多媒体(视频/语音/照片) | **送达时间**:演示设"2 分钟后"而非 15 天 |
| 选日期 ≥15 天 | 寄给别人 | 真实 10 年存储的耐久性 |
| 邮箱 OTP 登录 | 任何"我的信件"列表 | GDPR 删除/导出 |
| 封存动效(火漆→宝盒→消失) | 防刷(Turnstile)| 防滥用 |
| 送达:到期发邮件 | 重录 / 反悔撤回 | 多设备同步 |
| 看信页(网页,缓缓展开) | 付费收款 | |

**为什么这样切**:黑客松评委要看的是"魔法闭环 + 产品灵魂",不是生产级耐久。
上面"做"的 6 件,恰好能现场演完整个故事;"假装"的几件,只是把"等 10 年"压成"等 2 分钟"。

---

## 3. 用户旅程(逐屏 + 真实英文文案)

### 屏 1 · 写信
- 一块米黄色空白纸,光标已就位。
- Placeholder:`Dear future me…`
- 纯文字,随便改。

### 屏 2 · 选日期(同屏底部)
- 标签:`When should it find you again?`
- 最早提示:`No sooner than {date} · {15} days out`
- 比 15 天近的日子变灰、选不了。

### 屏 3 · 封存(第一次:验证邮箱)
- 按钮:`Seal`
- 第一次封存弹出:`Where should it be sent, years from now?`
- 输入邮箱 → 收到 6 位码 → 输码 → 验证通过。
- 这个邮箱 = 你的账号 + 将来放生的地址。

### 屏 4 · 封存仪式(动效)
- 火漆滴下、印章压下 → 信卷起放进时光宝盒 → **宝盒连信一起淡出消失**。
- 一句话:`Some things you keep by letting go.`
- 节奏要**慢**,有呼吸。

### 屏 5 · 封存之后(信消失了)
- 屏幕回到空白写信纸。
- 没有列表、没有计数、没有红点、没有"在路上"。
- 你唯一能做的:`再写一封`。

### 屏 6 · 那一天(邮件 + 看信页)
- 一封邮件到你邮箱:
  - 发件人显示名:`You, in {year}`
  - 主题:`A letter from you, {year}`
  - 顶部:`{N} years ago, you sealed something for this exact day.` + `Open it →`
  - 底部:**信的纯文字全文**(保命副本,链接打不开字也丢不了)。
- 点 `Open it →` → 看信页:火漆裂开、信纸一寸寸展开,慢慢地。

---

## 4. 两个仪式 = 灵魂(别敷衍)

**封存(app 内)**:火漆 → 宝盒 → 消失。情绪是"放手交给时间",**不是**"存进收藏夹"。
宝盒是动词,不是名词——绝不能变成以后能打开、能看"我攒了几封"的地方(那是列表换皮,破第一戒)。

**重逢(网页)**:唯一做"加法"的地方。craftsmanship 在这里燃烧——字体、配乐、拆封节奏。
节奏的灵魂是 **慢**,和外面那个快世界正好相反。

---

## 5. 功能需求(逐条,可当验收清单)

- **F1** 写信:纯文字,多行,封存前可改。
- **F2** 选日期:`deliver_on >= today + 15d`,**前端拦 + 后端拦**(演示模式可临时放开成分钟级)。
- **F3** 登录:邮箱 + OTP,无密码;设备保持登录,之后写信不再验证。
- **F4** 封存:写进数据库 → 锁死(不可改、不可看)→ 从 app 消失。
- **F5** 信的主人 = 账号 ID(`auth.uid()`),**不是邮箱字符串**(邮箱会变;送达那刻再查邮箱)。
- **F6** 送达:定时任务查"今天到期且没送过"→ 发邮件 → 回填 `delivered_at`(幂等,不重发)。
- **F7** 看信页:唯一、不可猜的链接,无需登录直接打开,缓缓展开。

## 6. 数据模型(最小)

```
letters
  id            主键 (uuid)
  owner_id      uuid  = auth.uid()  (身份,稳定)
  body          信的内容(静态加密存储)
  sealed_at     封存时间
  deliver_on    送达日 (date)
  delivered_at  送达后回填;null = 未送达(防重发)
  reveal_token  看信页的不可猜链接 token
```

门规则(RLS = Row Level Security = 行级安全,数据库自带的门卫):
- 只允许 `insert`,且 `owner_id = auth.uid()`。
- **不开** select / update / delete —— app 永远读不到信(同构第一戒)。

## 8. 五戒 = 硬性验收

1. **封存即消失** → 封存后无任何列表/计数/红点/"在路上"。
2. **真实高于体面** → V1 纯文字,但架构不挡将来多媒体。
3. **活得比公司久** → 信的全文永远躺在用户自己邮箱正文里;看信页可消失,绝不是唯一副本。
4. **为神圣时刻收费** → 黑客松不收款,但不得加广告/打扰。
5. **小而美** → 不加任何"提升日活"的功能。

## 9. 台上演示脚本(目标 90 秒)

1. 打开 app,当场写一句真心话。(10s)
2. 选日期——演示模式:`2 minutes from now`。(5s)
3. 按 `Seal` → 输邮箱 → 输码 → **火漆封印 → 宝盒消失**。(20s)
4. 屏幕空了:"看,它从我手里消失了。没有列表,没有红点。"(10s)
5. 切到邮箱,等那 2 分钟(或提前埋一封)。**邮件到了**,发件人写着 `You, in 2026`。(15s)
6. 点 `Open it →` → 看信页缓缓展开,读出那句话。(20s)
7. 收尾:"这就是写给未来的自己——你把此刻交给时间,时间替你保管,到那天还给你。"(10s)

## 10. 两天工时计划(粗排)

**Day 1 — 闭环跑通(功能优先,丑也行)**
- 上午:登录界面(邮箱→收码→输码→验证)+ 设备保持登录。
- 下午:收紧数据库(owner=账号 ID)+ 封存接到登录的人。
- 傍晚:加演示模式开关(日期下限临时放开到分钟);跑通"封存→写进库"。

**Day 2 — 魔法 + 灵魂(让人心头一颤)**
- 上午:送达定时任务(查到期→发邮件,带看信页链接)。先手动触发跑通,再上 cron。
- 下午:看信页(网页)——拆封动效,慢节奏展开。
- 傍晚:封存仪式动效(火漆→宝盒→消失)+ 串一遍演示脚本 + 录个备用 demo 视频(怕现场网络/邮件延迟)。

**留 buffer**:动效是最容易超时的,排在最后;就算来不及,闭环(写→封→送→看)必须先通。
